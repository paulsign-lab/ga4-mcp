# Google Cloud 인증 설정 가이드

> **setup.sh 실행 전** 이 가이드를 먼저 완료하세요.  
> 소요 시간: 약 10~15분 · 한 번만 하면 됩니다.

## 이 작업이 필요한 이유

Claude Code가 여러분의 GA4 데이터를 읽으려면,  
Google에게 "이 프로그램이 내 GA4에 접근해도 된다"는 **허가증**이 필요합니다.  
그 허가증을 만드는 과정이 바로 이 가이드입니다.

> ⚠️ **어떤 Google 계정을 쓸지 먼저 결정하세요**  
> GA4 데이터를 볼 수 있는 계정 (GA4 속성에 접근 권한이 있는 계정) 으로  
> 이 가이드 전체를 진행해야 합니다.  
> GCP 콘솔과 GA4 로그인 계정이 **반드시 동일**해야 합니다.

---

## 전체 5단계 흐름

```
1단계: GCP 프로젝트 만들기          (2분)
2단계: Analytics Data API 활성화    (1분)
3단계: OAuth 동의 화면 설정          (4분)
4단계: OAuth 클라이언트 ID 만들기    (2분)
5단계: JSON 파일 다운로드 & 저장     (1분)
         ↓
   setup.sh 실행 → 자동 완료
```

---

## 1단계 · GCP 프로젝트 만들기 (2분)

1. [console.cloud.google.com](https://console.cloud.google.com) 접속
2. **GA4에 접근 권한이 있는 Google 계정**으로 로그인
3. 상단 왼쪽 프로젝트 이름 클릭 → **새 프로젝트**

   ```
   프로젝트 이름: ga4-mcp
   위치: 조직 없음 (기본값)
   ```

4. **만들기** 클릭
5. 오른쪽 상단 종 모양(🔔) 알림에 "프로젝트 생성됨" 나타나면 완료

> 💡 이미 GCP 프로젝트가 있다면 새로 만들지 않고 기존 것을 선택해도 됩니다.

---

## 2단계 · Google Analytics Data API 활성화 (1분)

1. 상단 검색창에 `Google Analytics Data API` 입력 후 Enter
2. 검색 결과에서 **Google Analytics Data API** 클릭

   > ⚠️ "Google Analytics Admin API" 와 헷갈리지 마세요. **Data API** 입니다.

3. 파란색 **사용 설정** 버튼 클릭
4. 체크 표시(✓)와 함께 "API가 사용 설정됨"이 나타나면 완료

---

## 3단계 · OAuth 동의 화면 설정 (4분)

이 단계가 가장 중요합니다. **테스트 사용자에 본인 이메일을 추가하는 것**을 빠뜨리는 분이 많습니다.

### 3-1. 동의 화면 시작

1. 왼쪽 메뉴 → **API 및 서비스** → **OAuth 동의 화면**
2. User Type: **외부** 선택

   > 💡 **왜 외부인가요?**  
   > "내부"는 Google Workspace 조직 내 사용자만 인증 가능합니다.  
   > 개인 Gmail이나 회사 계정 모두 "외부"로 해야 차단 없이 인증됩니다.

3. **만들기** 클릭

### 3-2. 앱 정보 입력

| 항목 | 입력값 |
|---|---|
| 앱 이름 | `ga4-mcp` |
| 사용자 지원 이메일 | 본인 이메일 선택 |
| 개발자 연락처 이메일 | 본인 이메일 입력 |

나머지는 비워두고 **저장 후 계속** 클릭

### 3-3. 범위 (스코프) — 건너뛰기

아무것도 추가하지 않고 **저장 후 계속** 클릭

### 3-4. 테스트 사용자 추가 ⚠️ 핵심 단계

**이 단계를 빠뜨리면 인증 시 "액세스 차단됨" 오류가 납니다.**

1. **ADD USERS** 버튼 클릭
2. 본인 이메일 주소 입력 (GA4에 접근하는 계정)
3. **추가** 클릭
4. 이메일이 목록에 나타나는지 확인
5. **저장 후 계속** 클릭

### 3-5. 요약 화면

내용 확인 후 **대시보드로 돌아가기** 클릭

---

## 4단계 · OAuth 클라이언트 ID 만들기 (2분)

1. 왼쪽 메뉴 → **API 및 서비스** → **사용자 인증 정보**
2. 상단 **+ 사용자 인증 정보 만들기** 클릭
3. **OAuth 클라이언트 ID** 선택

   | 항목 | 선택값 |
   |---|---|
   | 애플리케이션 유형 | **데스크톱 앱** ⚠️ |
   | 이름 | `ga4-mcp` |

   > ⚠️ **왜 데스크톱 앱인가요?**  
   > "웹 애플리케이션"으로 만들면 `setup.sh`에서 인증이 실패합니다.  
   > 반드시 **데스크톱 앱**을 선택하세요.

4. **만들기** 클릭
5. 팝업이 뜨면 닫고 다음 단계로

---

## 5단계 · JSON 파일 다운로드 & 저장 (1분)

1. 왼쪽 메뉴 → **사용자 인증 정보** (현재 화면)
2. 방금 만든 `ga4-mcp` 클라이언트 오른쪽 ↓ (다운로드) 아이콘 클릭
3. `client_secret_XXXX.json` 파일이 `~/Downloads/` 폴더에 저장됨

> ✅ **파일 이름을 바꾸지 마세요.**  
> `setup.sh`가 `client_secret_*` 패턴으로 자동 탐지합니다.

> ✅ **파일을 `~/Downloads/` 폴더에 그대로 두세요.**  
> `setup.sh`가 이 위치를 자동으로 찾습니다.

## 완료 체크리스트

setup.sh를 실행하기 전에 아래를 확인하세요.

```
☐  GCP 프로젝트 생성 완료
☐  Google Analytics Data API 활성화 완료
☐  OAuth 동의 화면 → User Type: 외부
☐  OAuth 동의 화면 → 테스트 사용자에 본인 이메일 추가 ⚠️

```

모두 체크됐으면 `ga4-mcp` 폴더로 돌아가 `bash setup.sh`를 실행하세요.

## 자주 묻는 질문

### Q. "Google Cloud" 를 처음 써봅니다. 비용이 드나요?

GA4 Data API는 개인·소규모 사용 기준 **무료**입니다.  
하루 수만 건 이상의 API 호출이 발생하지 않는 한 비용이 청구되지 않습니다.  
신용카드를 등록하지 않으면 비용이 청구되지 않습니다.

### Q. GCP 프로젝트가 이미 있는데 새로 만들어야 하나요?

아니요. 기존 프로젝트에 Analytics Data API만 추가로 활성화하면 됩니다.  
OAuth 동의 화면도 이미 설정돼 있다면 4단계(클라이언트 ID 만들기)부터 하면 됩니다.

### Q. 회사 Google Workspace 계정인데 "액세스 차단됨" 이 떴어요.

3단계에서 테스트 사용자에 본인 이메일이 추가됐는지 확인하세요.  
또는 회사 IT 관리자가 외부 OAuth 앱을 차단했을 수 있습니다.  
이 경우: GCP 콘솔 → OAuth 동의 화면 → "게시 상태"를 **프로덕션**으로 변경하거나  
IT 관리자에게 해당 클라이언트 ID의 접근 허용을 요청하세요.

### Q. `client_secret_XXXX.json` 파일이 여러 개 있어요.

가장 최근에 만든 파일 하나만 사용합니다.  
나머지는 삭제하거나 다른 폴더로 옮겨두세요.

### Q. 이 파일을 다른 사람과 공유하면 안 되나요?

맞습니다. **절대 공유하지 마세요.**  
이 JSON 파일은 개인 인증 정보가 포함된 민감한 파일입니다.  
`.gitignore`에 포함되어 있으므로 GitHub에는 업로드되지 않습니다.
