# 🚀 처음 시작할 때 — 내 GA4 환경 파악

GA4 MCP를 설치하고 처음 Claude에 접속했을 때 이 파일부터 시작하세요.  
어떤 데이터가 있는지 먼저 파악하면, 이후 분석 질문이 훨씬 정확해집니다.

---

## Step 1 · 연결 확인

Claude가 실행된 상태에서 아래 명령을 입력하세요.

```
/mcp
```

`ga4 · Connected` 가 보이면 연결 성공입니다.  
안 보이면 `cd ga4-mcp && claude` 를 다시 실행하세요.

---

## Step 2 · 기본 데이터 있는지 확인

```
지난 30일간 전체 세션 수와 활성 사용자 수를 알려줘
```

숫자가 나오면 정상 연결된 것입니다.

---

## Step 3 · 내 웹사이트 유형 선택

아래 중 해당하는 유형을 선택해 전용 프롬프트 파일을 사용하세요.

### 나는 어떤 유형인가?

| 유형 | 해당 조건 | 추천 파일 |
|---|---|---|
| **B2C 이커머스** | 온라인 쇼핑몰 / 제품 구매 기능이 있는 사이트 | `06-ecommerce-report.md` |
| **B2B 기업 웹사이트** | 제품 소개·카탈로그 다운로드·문의 폼이 있는 사이트 | `07-b2b-report.md` |
| **일반 마케팅 분석** | 채널·트래픽 분석이 주목적 | `01` ~ `05` 파일 |

---

## Step 4 · 내 전환 이벤트 확인

전환 분석을 제대로 하려면 GA4에 어떤 이벤트가 있는지 먼저 확인해야 합니다.

```
지난 28일 기준으로 어떤 전환 이벤트가 발생했는지 종류와 건수를 표로 보여줘
```

**B2C 이커머스라면** `purchase`, `add_to_cart`, `begin_checkout` 이 있어야 합니다.  
**B2B 사이트라면** `generate_lead`, `form_submit`, `file_download` 등이 있어야 합니다.

> 💡 이벤트가 없거나 건수가 0이면 GA4 전환 이벤트 설정이 안 된 것입니다.  
> GA4 → 관리 → 이벤트에서 주요 이벤트를 전환으로 표시해두세요.

---

## Step 5 · 채널별 기본 현황 파악

```
지난 28일 채널별 세션 수와 사용자 수를 표로 정리해줘.
어떤 채널에서 트래픽이 가장 많이 오는지 순위도 알려줘.
```
